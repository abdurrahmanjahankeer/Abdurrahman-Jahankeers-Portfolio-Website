**Project Structure**:

The site includes the required 4 HTML pages plus project sub-pages:

- homepage.html → The page, the viewer starts on with all the navigation boxes.
- aboutme.html → Includes intro, image, video with poster/controls, games info, resume (skills, experience, education, awards).
  - AJ about me.mp4, AJ about me poster.png and AJ web pic.png
- projects.html → List of projects with headings and short descriptions. Each project page includes the project name, its explanation and the partners I worked with.
- crs.html → Car Rental System
- tas.html → Text Autocompletion System
- yylvp.html → Ying Yang Leadership Video Project
- vpc.html → Video Presentation on Computers
- ntcp.html → Network Topology Configuration Project
- contactme.html → Contact form which includes full name, email and the message of the viewer. The form also includes validations.

Each page uses semantic tags (header, nav, footer, article, etc.) and includes a footer with:
- Contact email
- Copyright
- Social media links with image icons such as Instagram, Twitter, YouTube and Facebook.

**GUI / Interface**:

Responsiveness:

- mobile.css → max-width: 480px
- tablet.css → 481px – 959px
- laptop.css → min-width: 960px

- These width sizes were chosen to target different view ports such as smartphones, tablets, and laptop/desktop screens.
- I have used these dimensions, because these were the basic dimensions that we learnt from the teacher during the week 3 lecture slides.

Gradients:
- Applied a linear gradient background (black → navy → deep blue) to all pages (body).
- Applied an angled linear gradient (30deg, mediumseagreen → yellow) to all five project buttons on projects.html.

Color Scheme:
- I designed my color scheme:
  - Dark Blue (rgb(0,0,128))
  - Medium Sea Green
  - Orange (highlight/accent)
  - White (text)

**Testing and Validation**:
- HTML Validation: All HTML files tested with W3C HTML Validator - no errors
- CSS Validation: All CSS files tested with W3C CSS Validator – two errors. 1 was a list error and the other was a max-width error. Both these errors called “markdown body” were errors that I could not fix as I didn’t learn them in class yet.
- Link Checker: Verified with W3C Link Checker – all links valid.
- Spell Check: All text checked, no spelling errors.
- Accessibility: Tested with WAVE – No errors or contrast errors. 1 alert and 1 feature. 2 structural elements and 0 aria.
